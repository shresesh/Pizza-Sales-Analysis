# Pizza Sales Analysis: End-to-End Business Intelligence

## 🍕 Project Overview
This project provides a comprehensive analysis of pizza sales performance to identify key business indicators and growth opportunities. By leveraging **SQL Server** for data extraction and **Power BI** for visualization, the analysis monitors essential KPIs, identifies peak sales periods, and categorizes menu performance to drive data-driven decision-making.

## 📊 Key Indicators & Business Insights
* **Revenue & Volume:** Successfully tracked total revenue, average order value, and a total of **49,574 pizzas sold**.
* **Temporal Trends:** Identified that **Friday and Saturday evenings** exhibit the highest order volume, while **July and January** are peak months for sales.
* **Category Intelligence:** Discovered that the **Classic Category** contributes most significantly to total sales volume.
* **Product Performance:** Isolated the **Top 5 and Bottom 5 pizzas** by revenue, quantity, and total orders to optimize menu engineering.

## 🛠️ Technical Toolkit
* **SQL Server:** Developed complex queries to calculate metrics such as **Average Pizzas Per Order** and **Daily/Monthly sales trends**.
* **Power BI:** Built an interactive two-page dashboard featuring a **Home Page** for KPIs and a **Best/Worst Sellers Page** for product-level analysis.
* **Data Transformation:** Performed data cleaning and formatting of order dates and times to ensure temporal accuracy.

## 🖥️ Dashboard Features
* **Executive Summary:** High-level cards for Revenue, Average Order Value, and Total Pizzas Sold.
* **Trend Visualizations:** Daily and Monthly charts to pinpoint high-demand periods for staffing optimization.
* **Market Share Analysis:** Pie charts showing the distribution of sales by pizza category and size.
* **Product Leaderboard:** Dynamic bar charts showcasing the best and worst-performing menu items.

## 📂 How to Run
1. Execute the provided **SQL scripts** in SQL Server to generate the necessary tables and metrics.
2. Connect the **Power BI** file to your SQL Server data source.
3. Refresh the data to populate the interactive visualizations.
